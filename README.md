# Blog-Read-Forced

「後で読む」と保存した記事を結局読まない問題を解決するツール。
iPhone で気になった記事をワンタップで保存し、毎日12時にランダムで1件通知して**強制的に記事を読む習慣**を作る。

## セットアップ手順

### 1. Google スプレッドシートを作成

1. Google ドライブで新しいスプレッドシートを作成する
2. シート名を `articles` に変更する（GAS が自動作成することも可）

---

### 2. GAS をデプロイする

1. スプレッドシートのメニュー → **「拡張機能」→「Apps Script」** を開く
2. `Code.gs` の内容をエディタに貼り付けて保存する（Ctrl+S）
3. **「デプロイ」→「新しいデプロイ」** をクリック
4. 設定:
   - 種類: **ウェブアプリ**
   - 説明: 任意
   - 実行ユーザー: **自分**
   - アクセスできるユーザー: **全員**
5. **「デプロイ」** をクリック → 表示される **ウェブアプリ URL をコピー** して保存する

> ⚠️ コードを更新したときは「デプロイを管理」→「編集」→「新しいバージョン」を選んで更新すること（URL は変わらない）

---

### 3. Chrome 拡張をインストールする

1. アイコン画像（`icon16.png`, `icon48.png`, `icon128.png`）を `chrome-extension/` フォルダに配置する
2. Chrome で `chrome://extensions` を開く
3. 右上の **「デベロッパーモード」** を ON にする
4. **「パッケージ化されていない拡張機能を読み込む」** をクリック
5. `chrome-extension/` フォルダを選択する
6. ツールバーの拡張アイコンをクリックし、**GAS URL を入力して「URL を保存」** する

---

### 4. iPhone ショートカットを設定する

| ステップ | アクション | 設定内容 |
|---------|-----------|---------|
| ① | ショートカットの入力を受け取る | 入力タイプ: URL |
| ② | 変数を設定 | 変数名「pageURL」= 入力のURL |
| ③ | 辞書 | `url`: pageURL、`source`: "iPhone" |
| ④ | URLのコンテンツを取得 | URL: GAS URL、メソッド: POST、本文: 辞書(JSON) |
| ⑤ | 通知を表示 | 本文: 「登録しました」 |

Safari / Chrome の共有ボタンからこのショートカットを実行することで記事を登録できる。

## 使い方

### 記事を登録する

**iPhone から:**
Safari / Chrome で記事を開く → 共有ボタン → 作成したショートカットを実行

**Chrome から:**
記事ページで拡張アイコンをクリック → 「この記事を登録」ボタンをクリック

### 通知を受け取る

毎日12:00に Chrome がバックグラウンドで未読記事を1件ランダム取得し、デスクトップ通知を表示する。
通知をクリックすると記事が新しいタブで開く。

### 既読にする

スプレッドシートの `articles` シートを開き、該当行の D列（ステータス）を `既読` に変更する。
